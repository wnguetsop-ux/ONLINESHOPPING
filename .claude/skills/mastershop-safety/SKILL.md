# MasterShop Safety

Use this skill when editing WhatsApp logic, billing-sensitive flows, auth, pricing controls or automation.

## Non-negotiables

- No automatic WhatsApp sending by default
- AI can suggest, never send autonomously
- Future outbound flows must surface cost risk and 24h window constraints
- Keep merchant data isolated correctly
- Keep secrets on the server only

## Product safety lens

- Favor human validation over invisible automation
- Prefer safe defaults over aggressive productivity claims
- A confusing UI is also a safety risk

## Refactor warning signs

- Removing user feedback
- Hiding state transitions
- Making a paid action look free
- Adding automation with no visible approval step
